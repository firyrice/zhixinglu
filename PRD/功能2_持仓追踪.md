# 功能2:持仓追踪（实盘组合）

> 摘自《知行录 产品需求文档》v1.1 第6.2节

---

## 1. 功能定位

用户从"看一只股票"到"管理自己的组合"的关键跃迁。通过手动录入构建实盘组合，让用户每天清晰地看到自己的真实盈亏和持仓结构，建立对自己投资组合的客观认知。

**与"持仓快速诊断"的关系**：持仓快速诊断是一次性的截图分析，持仓追踪是持续性的组合管理。诊断是"拍个X光片"，追踪是"每天量血压"。两者互补，持仓追踪是更深度的留存工具。

## 2. 用户故事

> 作为一个有实盘持仓的散户，我希望在一个地方录入我关注和持有的股票，每天打开就能看到真实的盈亏金额和组合结构，而不是在券商APP里被各种推荐和弹窗干扰。我想安静地、客观地面对自己的持仓。

## 3. 入口设计

- **首页"我的持仓"卡片**：首页第二优先级区域，展示今日组合简报（总盈亏金额 + 总资产），点击进入完整持仓页

## 4. 组合构建流程

### 4.1 添加股票

**第1步：搜索股票**
- 输入股票代码（如600519）或名称（如"贵州茅台"），支持模糊搜索
- 搜索结果展示：股票名称 + 代码 + 当前价格 + 今日涨跌幅
- 复用单股分析的搜索能力，MVP仅支持A股

**第2步：录入持仓信息**

| 字段 | 类型 | 输入方式 | 必填 |
|------|------|---------|------|
| 股票 | 自动填充 | 上一步选择 | 是 |
| 持仓数量（股） | 数字 | 数字键盘 | 是 |
| 成本价（元/股） | 数字 | 数字键盘 | 是 |
| 买入日期 | 日期选择 | 默认今天 | 否 |
| 备注 | 文本 | 限100字 | 否 |

**设计要点**：
- 持仓数量必须为100的整数倍（A股最小交易单位），输入非法值时温和提示
- 成本价用于计算持仓盈亏，用户可随时修改（比如多次买入后手动调整均价）
- 同一只股票可以多次添加（分批买入），系统自动合并计算平均成本

**第3步：确认添加**
- 展示录入摘要：股票名称、数量、成本价、持仓市值（按成本价计算）
- 确认后加入组合

### 4.2 管理股票

- **修改持仓**：可随时修改持仓数量和成本价（适用于加仓、减仓后调整）
- **删除股票**：从组合中移除，二次确认"确定移除这只股票？历史盈亏数据将不再追踪"
- **添加关注**：支持添加"仅关注"的股票（持仓数量为0），用于观察但不计入组合盈亏

### 4.3 数据刷新机制

- **自动刷新**：交易日盘中（9:30-15:00）每5分钟自动刷新最新价格；收盘后使用收盘价，不再刷新
- **手动刷新**：页面顶部下拉刷新，或点击刷新按钮
- **非交易日**：使用最近一个交易日的收盘价，页面标注"数据截止：X月X日收盘"
- **刷新状态提示**：页面底部小字显示"最后更新：XX:XX"，不用闪烁或强提示

## 5. 核心数据展示

组合构建完成后，持仓页分为两个主要视图，通过顶部Tab切换：**实盘盈亏** | **实盘穿透**

### 5.1 实盘盈亏

#### 5.1.1 组合总览卡片（页面顶部，常驻）

| 数据项 | 说明 | 展示形式 |
|--------|------|---------|
| 总资产 | 所有持仓股票按最新价计算的总市值 | 大号等宽数字，如 ¥352,680.00 |
| 今日盈亏金额 | 今日所有持仓的盈亏金额之和（基于昨收价→最新价的变动 × 持仓数量） | 带正负号的金额，如 +¥2,340.00 或 -¥1,580.00 |
| 今日盈亏比例 | 今日盈亏金额 / 昨日总资产 | 百分比，如 +0.67% |
| 总盈亏金额 | 所有持仓按（最新价 - 成本价）× 数量 的总和 | 带正负号的金额 |
| 总盈亏比例 | 总盈亏金额 / 总成本 | 百分比 |

**视觉设计**：
- 总资产用大号等宽字体（IBM Plex Mono），是视觉焦点
- 盈亏金额和比例使用低饱和度涨跌色（涨：`#D97757`，跌：`#7A9B6E`），不闪烁
- 卡片背景米白，不用渐变或花哨装饰

#### 5.1.2 个股盈亏列表

组合总览卡片下方，按持仓市值从大到小排列每只股票：

**每只股票一行，展示**：

| 数据项 | 说明 |
|--------|------|
| 股票名称 + 代码 | 如"贵州茅台 600519" |
| 最新价 | 实时/收盘价 |
| 今日涨跌幅 | 该股票最近一个交易日的涨跌百分比，如 +2.35% 或 -1.12% |
| 今日盈亏金额 | （最新价 - 昨收价）× 持仓数量，如 +¥680.00 |
| 持仓市值 | 最新价 × 持仓数量 |
| 持仓盈亏 | （最新价 - 成本价）× 持仓数量 |
| 持仓盈亏比例 | （最新价 - 成本价）/ 成本价 |

**交互设计**：
- 点击某只股票，展开详情卡片，显示：持仓数量、成本价、买入日期、备注
- 详情卡片底部提供操作按钮："修改持仓" "查看深度分析 →"（跳转单股分析）"记录决策 →"（跳转决策日志）
- 列表支持按"今日涨跌幅""持仓市值""持仓盈亏"排序切换
- 仅关注（持仓为0）的股票在列表底部单独分组，标题"关注中"

#### 5.1.3 示例

> **我的实盘组合**
>
> 总资产：¥352,680.00
> 今日：+¥2,340.00（+0.67%）
> 总盈亏：+¥28,680.00（+8.85%）
>
> | 股票 | 最新价 | 今日涨跌 | 今日盈亏 | 持仓市值 | 持仓盈亏 |
> |------|--------|---------|---------|---------|---------|

### 5.2 实盘穿透

#### 5.2.1 功能说明

将用户的实盘持仓按不同维度进行穿透分析，以饼图形式直观展示组合的结构特征。穿透的权重依据是每只股票的**持仓市值**（最新价 × 持仓数量），而非股票数量。

#### 5.2.2 行业分布穿透

**数据来源**：基于申万一级行业分类（通过akshare获取每只股票的所属行业）

**展示形式**：
- **饼图**：每个扇区代表一个行业，面积按该行业的持仓市值占比
- 饼图使用低饱和度配色，不超过8种颜色，超过8个行业的合并为"其他"
- 饼图中心显示"行业分布"标题
- 每个扇区标注：行业名称 + 占比百分比

**饼图下方列表**：

| 行业 | 持仓市值 | 占比 | 包含股票 |
|------|---------|------|---------|
| 食品饮料 | ¥168,000 | 47.6% | 贵州茅台 |
| 电力设备 | ¥107,650 | 30.5% | 宁德时代 |
| 非银金融 | ¥77,030 | 21.9% | 中国平安 |

#### 5.2.3 股票风格穿透

**风格分类维度**：

| 风格维度 | 分类标准 | 分类 |
|---------|---------|------|
| 市值规模 | 基于总市值 | 大盘（>1000亿）/ 中盘（200-1000亿）/ 小盘（<200亿） |
| 估值风格 | 基于PE分位数 | 价值型（PE历史分位<30%）/ 均衡型（30%-70%）/ 成长型（>70%） |
| 股息特征 | 基于近12个月股息率 | 高股息（>3%）/ 中股息（1%-3%）/ 低股息（<1%） |

**展示形式**：
- **三个饼图**，分别对应市值规模、估值风格、股息特征三个维度
- 每个饼图按持仓市值占比划分扇区
- 饼图配色统一使用产品色系，三个饼图之间色系有区分但整体和谐

**示例**：

> **市值规模分布**
> - 大盘股：78.1%（贵州茅台、中国平安）
> - 中盘股：21.9%（宁德时代）
> - 小盘股：0%
>
> **估值风格分布**
> - 价值型：69.5%（贵州茅台、中国平安）
> - 成长型：30.5%（宁德时代）
>
> **股息特征分布**
> - 高股息：47.6%（贵州茅台）
> - 中股息：21.9%（中国平安）
> - 低股息：30.5%（宁德时代）


#### 5.2.4 穿透图表交互

- 点击饼图扇区，高亮对应的股票列表
- 饼图支持手势操作：点击扇区弹出详情浮层（包含的股票及各自市值）
- Web端支持hover显示详情

## 6. 数据计算规则

### 6.1 价格数据

| 场景 | 使用价格 |
|------|---------|
| 交易日盘中 | 最新实时价格（延迟≤5分钟） |
| 交易日收盘后 | 当日收盘价 |
| 非交易日 | 最近一个交易日收盘价 |

### 6.2 盈亏计算

- **今日盈亏金额** =（最新价 - 昨收价）× 持仓数量
- **今日盈亏比例** = 今日盈亏金额 / 昨日持仓市值
- **持仓盈亏金额** =（最新价 - 成本价）× 持仓数量
- **持仓盈亏比例** =（最新价 - 成本价）/ 成本价
- **组合今日盈亏** = 所有个股今日盈亏金额之和
- **组合总资产** = 所有个股（最新价 × 持仓数量）之和

### 6.3 边界情况处理

- **停牌股票**：使用停牌前最后一个交易日的收盘价，标注"停牌中"
- **退市股票**：标注"已退市"，市值计为0，提示用户移除
- **新股上市首日**：无昨收价时，今日盈亏基于发行价计算

|------|--------|-------|

## 11. 后续迭代方向（MVP不做）

- **历史净值曲线**：记录每日组合净值，生成类似基金的净值走势图
- **组合对比基准**：组合收益 vs 沪深300 / 中证500 的对比曲线
- **持仓变动时间线**：记录每次加仓/减仓操作，形成组合变动历史
