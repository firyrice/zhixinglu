# 功能3：巴菲特来信 — 产品设计方案

## 概述

每日收盘后，基于用户持仓数据，以巴菲特的口吻和价值投资思维框架，生成一封深度分析报告。帮助"没时间盯盘"的价值投资者一站式完成交易追踪和决策辅助。

**核心定位**：深度研报型 · 强人格化 · 巴菲特致合伙人信的日版

**MVP触发方式**：用户手动点击生成（后续迭代加入每日自动定时生成）

---

## 一、来信内容结构（5大模块）

### 信头

- 日期标识：`BUFFETT'S LETTER · 2026年5月4日 · 收盘总结`
- 标题：`致我的合伙人`
- 开场白：巴菲特口吻的个性化开场，引用当日组合表现数据，自然过渡到正文

### 模块一：今日持仓全景

**组合总览（数据卡片）**：
- 今日盈亏（金额 + 百分比）
- 总资产
- 持仓总盈亏（金额 + 百分比）
- 跑赢/跑输沪深300幅度

**个股逐一点评**（每只持仓股包含）：
- 当日表现：涨跌幅、成交量变化（放量/缩量百分比）、换手率
- 资金动向：主力资金净流入/流出金额、北向资金增减持
- 技术信号：关键均线位置、支撑/压力位、MACD/KDJ等信号
- 巴菲特点评：用价值投资视角解读当日走势，评估是否符合买入逻辑

> 示例："贵州茅台今天微跌0.3%，成交缩量，这恰恰是我喜欢看到的——说明浮筹在减少。当前PE 28倍，处于近5年40%分位，还不算便宜，但如果你的买入逻辑是基于其品牌护城河和提价能力，短期波动不应该困扰你。"

### 模块二：精选热点情报

筛选标准：**宁缺毋滥**，只选与持仓直接相关的、被广泛讨论的、可能对股价有重大影响的信息。每日精选 3-5 条。

每条信息包含：
- 影响等级标签：`重大`（红色）/ `关注`（金色）
- 关联个股
- 信息标题
- 来源与时间
- 巴菲特解读：分析该信息对股价的潜在影响，给出价值投资视角的判断

信息来源：公司公告、财经新闻（东方财富/同花顺/新浪财经）、券商研报摘要、行业政策文件。通过 LLM + Web Search 获取并筛选。

> 示例：[重大 · 关联：贵州茅台] "茅台集团宣布2026年直销渠道占比提升至40%"
> 巴菲特解读："直销占比提升意味着渠道利润回流，这对毛利率是实质性利好。但要注意经销商体系的稳定性——茅台的护城河不仅是品牌，还有遍布全国的经销网络。短期利好，长期需要观察执行效果。"

### 模块三：市场大势研判

从价值投资视角解读宏观环境，包含：
- **大盘走势**：上证/深证/创业板指数表现、成交额变化、市场情绪（涨跌家数比、涨停跌停数）
- **板块轮动**：今日领涨/领跌板块、与持仓相关板块的表现、板块资金流向
- **北向资金**：今日净流入/流出金额、连续流入/流出天数、重点加仓/减仓个股（与持仓交叉）
- **政策与外围**：当日重要政策信号、美股/港股联动、汇率变化等外围因素

> 示例："今天两市成交额回落到8000亿以下，这是近两周的新低。作为价值投资者，我反而觉得这是好事——市场降温意味着投机资金在退场，而真正的价值会在潮水退去后显现。沪深300目前PE 12.5倍，处于近10年30%分位，从长期配置角度看，这个位置不贵。"

### 模块四：组合风险体检

- **仓位集中度**：单股最大仓位占比、前3大持仓占比、是否超过安全阈值（单股>30%预警），带可视化进度条
- **行业暴露**：行业分布是否过于集中、同一行业持仓占比、行业相关性风险
- **个股止损预警**：亏损超过阈值的个股（如 -15%）、连续下跌天数、是否触及关键支撑位
- **整体回撤**：组合从最高点的回撤幅度、与大盘回撤对比

> 示例："我注意到你的组合里消费板块占了60%，这让我有些担心。虽然消费股是价值投资的经典标的，但过度集中在单一板块会让你在行业逆风时承受过大压力。建议考虑在银行或公用事业中配置一些'压舱石'。另外，中国中免已经亏损18%，接近我通常建议的止损线——不是说一定要卖，但你需要重新审视当初的买入逻辑是否还成立。"

### 模块五：策略建议

基于以上四个模块的分析，给出**务实、谨慎、具体**的操作建议。

每条建议包含：
- 操作方向标签：`持有`（绿色）/ `关注加仓`（金色）/ `考虑减仓`（红色）
- 关联个股
- 理由：简明扼要的分析依据
- 风险提示：该操作的潜在风险和注意事项

### 信尾

- 巴菲特风格的收尾寄语（价值投资哲学金句）
- 署名：`— 你的投资伙伴，沃伦`
- 免责声明：`以上内容由AI生成，仅供参考，不构成投资建议。投资有风险，决策需谨慎。`

## 二、交互设计

### 2.1 用户旅程

```
首页来信区域 → 点击「生成今日来信」→ 流式生成来信（边生成边阅读）→ 生成完毕存入信箱
                                                                    ↓
首页来信区域（未读卡片）→ 点击「阅读来信」→ 来信详情页 → 自动标记已读
                                                        ↓
                                          底部「查看历史来信」→ 信箱列表页
```

### 2.2 首页来信区域

位置：首页顶部，搜索栏下方，作为首页专属区域。

**状态A — 有未读来信**：
- 深墨绿渐变背景卡片，视觉醒目但不刺眼
- 展示：日期、标题「致我的合伙人」、来信开头2行预览、持仓股数量
- 右下角金色「阅读来信 →」按钮
- 点击整张卡片进入来信详情

**状态B — 无来信 / 已读**：
- 虚线边框空状态卡片
- 信封图标 + 文案「还没有今日来信」+「收盘后点击生成，巴菲特会为你写一封专属分析」
- 居中「生成今日来信」按钮（深墨绿底）
- 底部「查看历史来信 →」链接

### 2.3 来信详情页

路由：`#/letter/{id}` 或 `#/letter/latest`

**布局**：
- 顶部导航栏：左侧「← 返回」，右侧日期
- 内容区域：居中单列，最大宽度 720px（桌面端），全宽（移动端）
- 信头：居中排版，`BUFFETT'S LETTER` 小字标识 + 标题 + 金色分隔线
- 5个模块依次排列，模块间用 `· · ·` 分隔
- 信尾：分隔线 + 寄语 + 署名 + 免责声明

**流式生成体验**：
- 点击生成后直接进入详情页
- 内容流式渲染，逐段出现（复用现有报告的 SSE 流式机制）
- 生成过程中顶部显示「正在撰写...」状态
- 生成完毕后状态变为日期，来信自动存入数据库

**数据卡片**：
- 桌面端：4列网格
- 移动端：2×2网格
- 数字使用等宽字体（IBM Plex Mono），涨用淡红 `#D97757`，跌用淡绿 `#7A9B6E`

**个股点评卡片**：
- 白色圆角卡片，轻阴影
- 顶部：股票名称 + 代码 + 涨跌幅
- 中部：关键数据标签（成交量变化、主力资金、均线位置）
- 底部：巴菲特口吻的斜体点评

**策略建议卡片**：
- 白色圆角卡片，左侧彩色边框标识操作方向
- 绿色 = 持有，金色 = 关注加仓，红色 = 考虑减仓
- 包含操作标签、股票名称、理由、风险提示（灰底小字）

### 2.4 信箱列表页

路由：`#/mailbox`

入口：首页来信区域底部「查看历史来信」链接

**布局**：
- 顶部：标题「信箱」+ 统计（共X封 · X封未读）+ 返回首页链接
- 列表：按日期倒序排列

**信件卡片**：
- 未读：白色卡片 + 金色左边框 + 金色圆点 + 「未读」标签 + 较深文字 + 较强阴影
- 已读：白色卡片 + 无边框 + 透明度 0.85 + 较浅文字 + 较弱阴影
- 每张卡片展示：标题、来信开头2行预览、日期、持仓股数、当日组合涨跌幅
- 点击进入来信详情，自动标记已读

**删除操作**：
- 移动端：左滑显示删除按钮
- 桌面端：hover 显示删除图标
- 二次确认弹窗

## 四、数据架构

### 4.1 来信存储

复用现有 `app/models/history.py` 的 SQLite 模式，新建 `buffett_letters` 表：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PRIMARY KEY | 自增ID |
| date | TEXT | 来信日期（YYYY-MM-DD） |
| title | TEXT | 标题（默认"致我的合伙人"） |
| content | TEXT | 完整HTML内容 |
| summary | TEXT | 开头摘要（用于列表预览） |
| portfolio_snapshot | TEXT | 生成时的持仓快照（JSON） |
| daily_return | REAL | 当日组合收益率 |
| stock_count | INTEGER | 持仓股数量 |
| is_read | INTEGER | 0=未读, 1=已读 |
| created_at | TEXT | 创建时间 |

### 4.2 API 端点

| Method | Path | Purpose |
|--------|------|---------|
| POST | `/api/letter/generate` | 触发生成来信（SSE流式返回） |
| GET | `/api/letters` | 获取来信列表（分页，按日期倒序） |
| GET | `/api/letter/{id}` | 获取单封来信详情 |
| PUT | `/api/letter/{id}/read` | 标记来信已读 |
| DELETE | `/api/letter/{id}` | 删除来信 |
| GET | `/api/letter/latest` | 获取最新来信（首页通知用） |

### 4.3 生成流程

```
用户点击「生成今日来信」
  → 前端 POST /api/letter/generate（SSE连接）
  → 后端获取持仓数据（从前端传入 localStorage 持仓）
  → 后端并行获取：
      - 各持仓股实时行情（portfolio_data.py）
      - 各持仓股资金流向、技术指标（akshare）
      - 市场大盘数据（market_data.py）
      - 持仓相关新闻/公告（LLM + Web Search）
  → 按5个模块依次调用 LLM 生成内容
  → 每个模块生成完毕即通过 SSE 推送 HTML 片段
  → 全部生成完毕后存入 SQLite
  → 前端流式渲染，生成完毕更新状态
```

### 4.4 前端数据流

- 持仓数据：从 `store.js` 的 localStorage 读取，生成时作为请求参数传给后端
- 来信列表/详情：从后端 API 获取
- 未读状态：后端存储，首页加载时调用 `/api/letter/latest` 检查

## 五、AI Prompt 设计要点

### 人格设定

全程以巴菲特第一人称写作，核心特征：
- 称呼用户为"合伙人"
- 语言亲和、幽默、有智慧感
- 善用比喻和生活化的例子解释复杂概念
- 强调安全边际、护城河、长期复利等价值投资核心理念
- 对市场短期波动保持淡定，对基本面变化保持警觉
- 策略建议务实谨慎，绝不打鸡血，一切指向风险可控和长期可盈利

### Prompt 结构

每个模块独立 prompt，共享以下 context：
- 用户持仓列表（股票代码、名称、持仓数量、成本价、当前价、盈亏）
- 组合总览数据（总资产、今日盈亏、持仓盈亏）
- 当日市场数据（大盘指数、成交额、北向资金等）

各模块 prompt 额外注入对应的数据（个股行情、新闻、技术指标等）。

## 六、前端路由与文件规划

### 新增路由

| Hash | 页面 | 说明 |
|------|------|------|
| `#/letter/{id}` | 来信详情 | 阅读单封来信 |
| `#/mailbox` | 信箱列表 | 历史来信列表 |

### 边界情况

| 场景 | 处理方式 |
|------|---------|
| 用户无持仓数据 | 生成按钮置灰，提示「请先在持仓页添加股票」 |
| 当日已生成过来信 | 按钮文案变为「重新生成今日来信」，二次确认覆盖 |
| 生成过程中断网/关闭 | 已接收的内容保留在页面，提示「生成中断，请重试」 |
| 非交易日点击生成 | 正常生成，使用最近交易日的收盘数据，信中说明 |
| 持仓股数量很多（>15只） | 个股点评只详细分析前10只（按持仓市值排序），其余简要提及 |

### 新增/修改文件

| 文件 | 说明 |
|------|------|
| `app/static/js/letter.js` | 来信详情页逻辑（流式渲染、标记已读） |
| `app/static/js/mailbox.js` | 信箱列表页逻辑（列表加载、删除） |
| `app/static/index.html` | 新增首页来信区域、路由注册 |
| `app/static/js/router.js` | 注册新路由 |
| `app/ai/prompts.py` | 新增5个巴菲特来信模块的 prompt |

| `app/main.py` | 注册新 API 端点 |
